# UAV大飛機-航電組開發流程
環境:Ubuntu 24.04 + ROS 2 Jazzy

## 安裝 PX4 開發環境

```bash
cd
git clone https://github.com/PX4/PX4-Autopilot.git --recursive
bash ./PX4-Autopilot/Tools/setup/ubuntu.sh
cd PX4-Autopilot
make px4_sitl
```
## 安裝 ROS2 jazzy
```bash
sudo apt install software-properties-common
sudo add-apt-repository universe

sudo apt update && sudo apt install curl -y
export ROS_APT_SOURCE_VERSION=$(curl -s https://api.github.com/repos/ros-infrastructure/ros-apt-source/releases/latest | grep -F "tag_name" | awk -F'"' '{print $4}')
curl -L -o /tmp/ros2-apt-source.deb "https://github.com/ros-infrastructure/ros-apt-source/releases/download/${ROS_APT_SOURCE_VERSION}/ros2-apt-source_${ROS_APT_SOURCE_VERSION}.$(. /etc/os-release && echo ${UBUNTU_CODENAME:-${VERSION_CODENAME}})_all.deb"
sudo dpkg -i /tmp/ros2-apt-source.deb

sudo apt update
sudo apt upgrade -y
sudo apt install ros-jazzy-desktop
sudo apt install ros-dev-tools
echo "source /opt/ros/jazzy/setup.bash" >> ~/.bashrc
source ~/.bashrc
```

## 驗證 ROS 2 是否安裝成功

### Terminal 1

```bash
ros2 run demo_nodes_cpp talker
```
### Terminal 2
```bash
source /opt/ros/jazzy/setup.bash
ros2 run demo_nodes_py listener
```
## 安裝 Micro-XRCE-DDS-Agent v2.4.3

```bash
cd ~
git clone -b v2.4.3 https://github.com/eProsima/Micro-XRCE-DDS-Agent.git
cd Micro-XRCE-DDS-Agent
mkdir build
cd build
cmake ..
make -j$(nproc)
sudo make install
sudo ldconfig /usr/local/lib/
```
### 驗證binary在不在
```bash
MicroXRCEAgent --help
```
## 安裝PX4開發環境
```bash
cd ~
git clone https://github.com/PX4/PX4-Autopilot.git --recursive
bash ./PX4-Autopilot/Tools/setup/ubuntu.sh
```
*(重開機)*

## 建立 ROS 2 workplace
PX4 的 ROS 2 user guide 說明，ROS 2 端需要有跟 PX4 firmware 相容的 message 定義，所以要把 `px4_msgs` 放進 workspace；`px4_ros_com` 雖然現在不是必需，但官方也明講它仍然包含有用的 examples。
```bash
mkdir -p ~/ws_offboard_control/src
cd ~/ws_offboard_control/src

git clone https://github.com/PX4/px4_msgs.git
git clone https://github.com/PX4/px4_ros_com.git

cd ..
source /opt/ros/jazzy/setup.bash
colcon build
source install/local_setup.bash
```

## 運行SITL
PX4 文件說，模擬器會自動啟動 uXRCE-DDS client，並連到本機 UDP 8888；agent 則用 MicroXRCEAgent udp4 -p 8888 啟動。

啟動Agent
```bash
MicroXRCEAgent udp4 -p 8888
```
再開 PX4 SITL。官方文件對 Gazebo 的 x500 範例是：
```bash
cd ~/PX4-Autopilot
make px4_sitl gz_x500
```
如果 agent 跟 client 接上了，PX4 console 會出現 `uxrce_dds_client` 建立 data writer 的訊息。

## 試試Example
