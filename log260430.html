說明

繼上回4/23的目標，我們繼續進行offboard mode的開發。這次我們主要完成了offboard control的基礎架構和功能實現。

上回4/23目標

  • sensor_combined_listener 能穩定收到 /fmu/out/sensor_combined，而且訂閱時要用rmw_qos_profile_sensor_data，因為 PX4 publisher 的 QoS 跟 ROS 2 預設 subscriber QoS 不相容。
  • offboard_control 官方範例可以正常起飛到 5 m 並保持。官方範例本身就是用來驗證 ROS 2 ↔︎ PX4 ↔︎ uXRCE-DDS 這條鏈有通。

編寫 offboard mode 程式碼

複製官方offbaoard範例 做成自己的package

cd ~/ws_offbaord_control/src
ros2 pkg create --build-type ament_cmake my_offbaord_cpp --dependencies rclcpp px4_msgs

offboard mode 結構

檔名:src/examples/offbaord/offboard_control.cpp 類別:class OffboardControl: public rclcpp::Node main()裡面只做rclcpp::init()、spin()、shutdown()

node 介紹

三個publisher:

  • /fmu/in/offboard_control_mode
  • /fmu/in/trajectory_setpoint
  • /fmu/in/vehicle_command

重要成員

  • timer_
  • offboard_setpoint_counter_

建構子 OffboardControl() 在做什麼

  • 建立三個publisher
  • 把offboard_setpoint_counter_設成0
  • 建立一個100 ms timer

timer callback

流程幾乎都在callback裡

auto timer_callback = [this]() - > void {
    if (offboard_setpoint_counter_ == 10) {
        publish_vehicle_command(...切到 offboard...);
        arm();
    }
    publish_offboard_control_mode();
    publish_trajectory_setpoint();

    if (offboard_setpoint_counter_ < 11) {
        offboard_setpoint_counter_++;
    }
}
